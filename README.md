# 英語学習サポートAI（スプレッドシート連携）

ご要望の仕様に基づく、Google Apps Scriptを用いた英語学習チャットアプリの実装とセットアップ手順です。

## 1. アプリでできること（主な機能）

- **AIとの自然な英会話・質問**: 英語のフレーズ、文法、ニュアンスの違いなど、学習中の疑問をAIに気軽に質問し、親切な解説をもらうことができます。
- **重要データの自動抽出**: あなたの質問と解説の中から、「英語表現」「日本語訳」「例文」「ニュアンス」などの重要な構成要素をAIが自動で整理・抽出します。
- **スプレッドシートへの自動蓄積**: 抽出されたデータは、裏側で自動的にGoogleスプレッドシートのリストとして保存されていきます。あなた専用の「単語帳・フレーズ集」が質問するだけで勝手に出来上がります。
- **マルチデバイス対応**: スマホ、タブレット、PCのどのブラウザからでも見やすい画面設計で、スキマ時間にいつでもどこでも学習をスタートできます。

## 2. このアプリが活用できるシーン

- **海外ドラマや映画のセリフの解読**
  字幕やスクリプトで気になった「生きた表現」やスラングの細かいニュアンスを質問して掘り下げ、自分だけのフレーズ集を作成します。
- **英文メール・SNS発信のサポートと学習**
  「こういう場面でこう言いたい場合、自然な英語表現はある？」と質問し、実用的なビジネス表現やカジュアル表現のストックを増やします。
- **資格試験（TOEICや英検など）の語彙力強化**
  市販の単語帳でなかなか覚えられない単語について、コアイメージや似た単語とのニュアンスの違いを質問。スプレッドシートを自分専用の暗記帳として育てます。
- **日々の「独り言英会話（Soliloquy）」のお供に**
  「“面倒くさい”って英語でなんて言うんだろう？」など、日常のちょっとした疑問をいつでもスマホやPCから壁打ちして言語化します。

2. 作成したスプレッドシートのメニューから、「拡張機能」>「Apps Script」をクリックし、エディタを開きます。

   - エディタ左側のファイル一覧にある「＋」ボタンをクリックし、「HTML」を選択します。

### 2-3. スクリプトプロパティへの設定（Gemini APIキーの登録）
1. GASエディタの左側メニュー（歯車アイコン）から「プロジェクトの設定」を開きます。
2. 「スクリプト プロパティ」のセクションまでスクロールし、「スクリプト プロパティを追加」をクリックします。
3. 以下の通り入力します：
   - プロパティ: `GEMINI_API_KEY`
   - 値: （取得したGemini APIキーを貼り付け）
4. 「スクリプトプロパティを保存」ボタンをクリックします。

### 2-4. デプロイ設定
1. エディタ右上の「デプロイ」>「新しいデプロイ」をクリックします。
2. 「種類の選択」の歯車アイコンをクリックし、「ウェブアプリ」を選択します。
3. 以下の設定がされていることを**必ず確認**してください：
   - 説明: 英語学習アシスタント（任意）
   - 実行するユーザー: **「自分」**
   - アクセスできるユーザー: **「自分のみ」**
4. 「デプロイ」ボタンをクリックします。
5. アクセス承認が求められた場合は、「アクセスを承認」>対象のGoogleアカウントを選択>「詳細」>「◯◯（安全ではないページ）に移動」>「許可」の順に進めます。
6. 発行された「ウェブアプリのURL」にアクセスして使用を開始します。

## 4. アプリの仕組み（動作フローとコード解説）

このアプリは、すべて作成者のGoogleアカウント内で完結する **完全無料・保守ゼロ** のアーキテクチャで構成されています。裏側で「どのファイル」の「どの処理」が動くかを順を追って解説します。

### ① UIの表示とユーザー入力 (フロントエンド: `index.html`)
- **画面描画**: アプリのURLにアクセスすると、バックエンド（`main.gs`）の `doGet(e)` 関数が呼ばれ、フロントエンドの画面として `index.html` が表示されます。デザインにはTailwind CSSを利用しています。
- **入力アクション**: `index.html` 内の `sendMessage()` 関数が起動し、テキストボックスに入力されたユーザーのテキストを取得します。
- **バックエンド通信**: その後、JavaScriptから `google.script.run.withSuccessHandler(...)` を用いることで、画面を再読み込みすることなく（非同期に）バックエンドの `processChat(text)` へメッセージを送信します。

### ② Gemini APIへのリクエスト (バックエンド: `main.gs`)
- **プロンプト生成**: `main.gs` の `processChat(message)` が呼び出され、内部で `callGeminiApi(userMessage)` に処理を回します。
- **JSONスキーマの指定**: ここでの最大のポイントは、単語の意味を聞くだけでなく `responseSchema`（JSONスキーマ）を設定してAPIに送っている点です。これにより、「ただの文章」ではなく、「確実に指定した形式のJSONデータ」を返すようGeminiに要求しています。
- **API通信実行**: GASに組み込まれた `UrlFetchApp.fetch(...)` を使用し、最新の `gemini-2.5-flash` モデルへリクエストを飛ばします。

### ③ AIによる生成とデータ抽出 (Gemini API)
- Geminiは要求通り、以下の2つの要素を持ったJSONを生成して返却します。
  1. `reply_message`: チャットUIに表示するための、自然な日本語での回答文
  2. `extracted_data`: スプレッドシートに保存するために抽出された構造化データ（フレーズ、日本語訳、ニュアンス、例文、ジャンルタグなど）

### ④ スプレッドシートへの自動保存 (バックエンド: `main.gs`)
- **シートの取得と追記**: AIの生成完了後、`main.gs` 内の `saveToSheet(extractedData)` 関数が呼び出されます。
- `SpreadsheetApp.getActiveSpreadsheet()` でこのスクリプトに紐づいたシートを取得し、新しいIDやタイムスタンプを付与した上で、先ほどの生成データを最終行に1行（`sheet.appendRow`）として追記・蓄積します。

### ⑤ 画面への回答表示 (フロントエンド: `index.html`)
- **UIへの反映**: スプレッドシートの保存が無事完了すると、`processChat` は `reply_message` のみをフロントへ返還します。
- 待機していた `index.html` 側の `withSuccessHandler` がテキストを受け取り、最後に `appendAiMessage` 関数によって、画面上にAIの吹き出しとして結果が表示されます。
